Motivation

Patch generation is only the last mile.

In real repositories, coding agents first need to locate the right context: implementation files, tests, review constraints, and root-cause source files. End-to-end agent scores often hide whether failures come from bad generation or bad retrieval.

Agentic retrieval asks for the files an agent must read to finish the task, not merely the code most semantically similar to a query.

Indirect signals

PRs, comments, and traces rarely name all target files directly. The useful context is often one relation away.

Graph structure matters

Source/test links, package boundaries, symbol references, and failure paths carry signals that pure embedding similarity can miss.

Leakage is easy

Gold paths, basenames, raw patches, and final diffs can turn hard retrieval into trivial path matching. V1 filters these signals aggressively.

Benchmark Tasks

Three retrieval surfaces from real coding workflows.

Each sample is evaluated against the repository at its base commit. Fixed code is not indexed.

01 · code2test

Implementation intent → related tests

106samples

Review comment → extra context

51samples

Failure trace → root-cause source

Representative Examples

The query is often indirect.

These are simplified patterns observed during V1 construction. They show why agentic retrieval is not just semantic code search.

PR intent does not name tests

“Add support for optional values in shell completion.”

A semantic retriever may return completion implementation files. The agent may actually need shell-specific completion tests and snapshots. This requires source→test mapping, not just text similarity.

The reviewed file is not enough

“Should this be sanitized consistently with the web response path?”

The commented file is already known. The useful context may live in a web extension, existing endpoint tests, or a policy module. V1 scores the extra context, not the given file.

Failure traces point at symptoms

FAIL: TestOptionalCompletion · expected candidates, got different output

The trace may mention only a test file. The root cause can be in parser, runtime, or completion source code. V1 evaluates retrieval of source root causes, not trace-frame lookup.

Qwen wins code2test

Qwen3-Embedding-4B reaches 0.3225 MRR and 0.7230 Recall@20 on implementation-to-test retrieval.

Jina wins comment2context

Jina reaches 0.3282 MRR on review-context retrieval, narrowly ahead of Qwen by MRR.

RepoMap wins trace2code

Vectorless RepoMap reaches 0.2750 MRR and 0.8064 Recall@20 on failure-trace root-cause retrieval.

The trace task rewards structured failure-log and repo-graph retrieval. Weak embedding scores there are a benchmark signal, not a data bug.
